SikshaSetu — Rural Education Platform

Jan 2025 – Apr 2025 · Full-stack developer
View on GitHub →
Problem

Rural and underserved colleges run academic operations — attendance, timetables, assignments, notes, results, events, and classes — across disconnected manual systems. SikshaSetu unifies these into a single portal with secure role-based access for Students, Faculty, and Admins, including support for live virtual classrooms where physical infrastructure is limited.

Approach

Backend built on Java 17 and Spring Boot 3.5, with Spring Security and JWT enforcing role-based access across 3 roles and all API endpoints. Modeled a 10+ table relational schema with Spring Data JPA and Hibernate on MySQL (H2 for tests). Added an AI Assistant using LangChain4j that streams responses, answers portal-navigation questions, calls backend tools for data instead of guessing, and restricts admin-only data. Notes search uses a Qdrant vector database with Apache Tika for document text extraction (RAG). Live virtual classrooms run on ZEGOCLOUD, with video tokens generated server-side. Frontend is React 18 with Tailwind CSS and the ZEGOCLOUD UI Kit.

Java 17Spring BootSpring Security / JWT Spring Data JPAMySQLLangChain4j QdrantReactZEGOCLOUD

SahayLoan — Micro-Loan Platform

Jan 2026 – Present · Developer
View on GitHub →
Problem

Underserved communities in India often lack access to small, fast credit. Manual loan review is slow and inconsistent. SahayLoan lets users apply for micro-loans up to ₹1,00,000 from a phone, with automated credit decisions, digital KYC, and online EMI repayment, across three roles: Borrower, Admin, and Loan Provider.

Approach

Flutter frontend (Android, iOS, Web, Windows) talking to a FastAPI Python backend over REST. Loan eligibility is automated with a Random Forest classifier trained on 19 of 28 borrower attributes, reaching 76% test accuracy. KYC is automated with Tesseract OCR for Aadhaar and PAN extraction, removing manual data entry. Stripe handles multi-installment EMI payments. Firebase Auth and Firestore power the 3-role access system with real-time loan-status updates and push notifications. The loan workflow moves from application through admin review, approval or provider sharing, disbursement, and EMI repayment to completion.

FlutterFastAPIPython Scikit-learnFirebaseTesseract OCR Stripe

PhonePe Insights — ETL & Analytics Dashboard

Jan 2026 – Mar 2026 · Data engineering & dashboard developer
View on GitHub →
Problem

Raw PhonePe transaction data spread across states and categories is hard to query or interpret in its original form. The goal was to turn it into a structured, queryable dataset and a dashboard that surfaces transaction trends, regional breakdowns, and payment-category insights.

Approach

Built a Python ETL pipeline that extracts, cleans, and loads 9 raw transaction datasets — 50,000+ records across 36 states — into a normalized 5-table MySQL schema, reducing query time. SQLAlchemy and PyMySQL handle parameterized SQL queries for ad-hoc analysis. The result is delivered as a Streamlit dashboard with 10+ interactive Plotly charts covering trends, regions, and payment categories.
